## 第一章 绪论

### 1.1.1 基本概念

- **数据结构** 相互之间存在一种或者多种特定关系的数据元素的集合

### 1.1.2 数据结构三要素

- 数据的 **逻辑结构**
- 数据的 **存储结构**

    - 有穷性

## 第二章 线性表

### 2.2.1 线性表的定义

- 具有 **相同** 数据类型的n个数据元素的 **有限序列**
- 线性表是一种逻辑结构，而顺序表和链表指的是存储结构

### 2.2.2 顺序表示

- 可以进行随机访问，在O（1）时间内找到指定的元素
- 存储密度高，每个节点都只存储数据元素
- 插入和删除要移动大量的元素，插入操作平均要移动 $\frac n2$ 个元素，而删除操作平均要移动 $\frac {n-1}2$ 个元素

### 2.3.1 单链表的定义

- 单链表也是线性表，其不支持存取结构
- 双链表其实就是又有prev指针又有next指针
- 循环链表的含义就是链表的逻辑结构是一个环，不停地走next和prev路径会循环下去
- 难以估量线性表的长度或者存储规模的时候，就不要用顺序表

### 3.4.3 特殊矩阵的压缩存储

- 对称矩阵：对于数据位置 $a[i,j]$ 其存储位置（行优先）
  
    $$
    k=\begin{cases}
    \frac{i(i-1)}{2}+j-1\ \ when\ i \ge j\\
    \frac{j(j-1)}{2}+i-1\ \ when\ j \ge i\\
    \end{cases}
    $$
    
- 下三角矩阵：

$$
k=\begin{cases}
\frac{i(i-1)}{2}+j-1\ \ when\ i\ge j\\
\frac{n(n+1)}{2}\ \ when\ i < j
\end{cases}
$$

- 上三角矩阵：

$$
k=\begin{cases}
\frac{(i-1)(2n-i+2)}{2}+(j-1)\ \ when\ i \le j\\
\frac{n(n+1)}{2}\ \ when\ i > j
\end{cases}
$$

- 三对角矩阵（带状矩阵）：

$$
k=2i+j-3
$$

## 第五章 树与二叉树

### 5.1.2 基本术语

- 节点的度指的是它的孩子的个数
- 树的度指的是其中节点的最大度数
- 在王道当中，**根节点定义为第1层**，~~而不是第0层~~

### 5.1.3 树的性质

- 树的节点数n等于所有节点的度数之和+1（每一个节点都反应在其父节点的度数当中，除了根节点，也就是+1）
- 高度为h的m叉树最多有 $\frac{(m^h-1)}{(m-1)}$个节点

$$
total=1+m+m^2+\dots+m^{h-1}=\frac{m^h-1}{m-1}
$$

- 由上面的式子也可以知道，如果一个树度为m，节点数量为n，则求其最小高度h有：

$$
\begin{aligned}
\frac{m^h-1}{m-1} &\ge n\\
m^h-1 &\ge n(m-1)\\
m^h &\ge n(m-1)+1\\
h_{min}&=ceiling[\log_m{(n(m-1)+1)}]
\end{aligned}
$$

- 度为m，具有n个节点的树的最大高度h是n-m+1（只有一层是m个孩子，其他都按1个子节点算）

### 5.2.1 二叉树的定义和特性

- 二叉树不等于度为2的树，因为度为2的树至少有3个节点，而二叉树可以为空
- 如果把根节点编号为1，任意一个节点的父节点的编号是 $floor(\frac{i}{2})$，其左子节点的编号是 $i\times2$，而右节点编号是 $i \times 2 + 1$
- 满二叉树是全部都占满了，即有 $2^h-1$ 个节点
- 完全二叉树是从 1 ~ n 之间没有断层的二叉树
- 对于完全二叉树：
    - 如果 $i \le floor(\frac n2)$，那么这个节点就是分支节点，否则就是叶节点
    - 叶节点只有可能在最后两层出现
    - 如果有度为1的节点，只有可能有一个，而且只有左子节点（即最后一层放了一个叶节点的情况）
    - 如果有一个节点i为叶节点或者只有左孩子，那么之后的节点一定都是叶节点
    - 如果n是一个奇数，那么所有分支节点就左右子节点都有（因为除去根节点就是偶数个），如果n是一个偶数，那么编号最大的分支节点只有左子节点
    - 节点i的深度是 $floor(\log_2{(i)})+1$
- 二叉排序树：左子树上的所有节点小于根节点，右子树上所有节点大于根节点
- 平衡二叉树：任意一个节点的左右子树高度差不超过1
- 正则二叉树：子节点数要么是0要么是2
    - （对于正则k叉树定义类似，要么没有任何子节点，要么子节点填满）
- **非空二叉树上叶节点的数量等于度为2的节点数+1 （非常重要）**
  
    解释：
    
    $given:\\
    n=n_0+n_1+n_2\\
    n=n_1+2n_2+1\\
    that: n_1+2n_2+1=n_0+n_1+n_2\\
    implies:n_2+1=n_0$
    

### 5.4.3 树和森林的遍历

- 树的先根遍历和后根遍历等价于其对应的二叉树的先序遍历和中序遍历
- 王道上面对森林的中序遍历（相对于其一棵树而言，因为对其后序遍历等价于对其对应的二叉树进行中序遍历），实际上等价于后序遍历

## 第六章 图

### 6.1.1 图的定义

- 简单图是不存在重复边，不存在顶点到自身的边的图
- 完全图是任意两个顶点之间都有边，共 $\frac{n(n-1)}2$（如果是有向图的话是有两条，两个方向各一条 即$n(n-1)$条）
- 强连通图的含义是在有向图当中，任意一对顶点之间都有路径（不一定是只走一次，可以有中转站）
- 生成树和生成森林是针对无向图而言的
- 顶点v1到v2之间的一条路径是指顶点序列，以v1开始，以v2结束
- 如果一个图有n个顶点，并且边数大于n-1，则一定有环
- 简单路径是不出现顶点重复
- 简单回路是除了起点和终点以外都不重复

### 6.2.3 十字链表

- 对于每一个节点包含：
    - 把自己作为终点的弧列表
    - 把自己作为起点的弧列表
- 对于每一弧包含：
    - 和自己同起点的弧列表
    - 和自己同终点的弧列表
- 一个图的十字链表表示不唯一

### 6.2.4 邻接多重表

- 和十字链表很像，只不过由于邻接多重表表示的是无向边，所以没有起点终点之分，每个节点里面只有一个弧列表

### 6.4.1 最小生成树

### 6.4.2 最短路径

- dijkstra（堆优化后的代码）

```csharp
class Node 
{
    public int Id { get; set; }
    public int Distance { get; set; } = int.MaxValue;
    public List<(Node node, int distance)> Neighbours { get; set; } = new();
    public bool FoundShortestPath { get; set; } = false;
}

void DoDijkstra(Node startNode) 
{
    PriorityQueue<Node, int> heap = new();
    startNode.Distance = 0;
    heap.Enqueue(startNode, 0);

    while (heap.Count > 0) 
    {
        Node currentNode = heap.Dequeue();
        
        // 跳过已确定最短路径的节点
        if (currentNode.FoundShortestPath) 
            continue;
        
        currentNode.FoundShortestPath = true;

        foreach ((Node neighbor, int edgeDist) in currentNode.Neighbours)
        {
            // 跳过已确定最短路径的邻居
            if (neighbor.FoundShortestPath) 
            {
                continue;
            }
            
            int newDistance = currentNode.Distance + edgeDist;
            if (newDistance < neighbor.Distance)
            {
                neighbor.Distance = newDistance;
                heap.Enqueue(neighbor, newDistance);
            }
        }
    }
}
```

- Floyd 算法执行k次后，所得的距离矩阵 $A^{(k-1)}[i][j]$是指从i到j中间顶点的序号不大于k-1的最短路径的长度

```csharp
#nullable enable
class Graph
{
    public int[,] Matrix { get; }

    public Graph(int size) {
        Matrix = new int[size, size];
    }
}

int[,] DoFloyd(Graph graph) {
    int size = graph.Matrix.GetLength(0);

    int[,] dist_old = new int[size, size];
    int[,] dist_new = new int[size, size];

    // copy the original matrix
    for (int i = 0; i < size; i++)
    {
        for (int j = 0; j < size; j++)
        {
            dist_old[i, j] = graph.Matrix[i, j];
        }
    }
    
    // everytime, try to insert a new point on the path
    // from (from) to (to)
    // make it like (from) -path-> (via) -path-> (to)
    for (int via = 0; via < size; via++)
    {
        for (int from = 0; from < size; from++) 
        {
            for (int to = 0; to < size; to++) 
            {
                if (from == to) {
                    continue;
                }

                if (dist_old[from, via] == int.MaxValue || 
                    dist_old[via, to] == int.MaxValue) {
                    continue;
                }

                checked
                {
                    dist_new[from, to] = Math.Min(dist_old[from, to], dist_old[from, via] + dist_old[via, to]);
                }
            }
        }
        // swap
        (dist_old, dist_new) = (dist_new, dist_old);
    }
    return dist_old;
}
```

### 6.4.4 拓扑排序

- DFS可以用于拓扑排序，其原理是，使用DFS完成对a的访问时，一定完成了a的所有后继节点的访问。（后继节点的完成时间会更早）

```csharp
class Node 
{
    public int Id { get; }
    public List<Node> Neighbours { get; } = [];

    public Node(int id)
    {
        Id = id;
    }
}

void TopoDFS(Node node,
    HashSet<Node> visited,
    Dictionary<Node, int> endTime,
    ref int time) 
{
    visited.Add(node);

    foreach (var neibour in node.Neighbours)
    {
        if (!visited.Contains(neibour))
        {
            TopoDFS(neibour, visited, endTime, ref time);
        }
    }

    endTime[node] = ++time;
}

IEnumerable<Node> TopoByDFS(Node node) 
{
    Dictionary<Node, int> endTime = [];
    HashSet<Node> visited = new HashSet<Node>();
    int time = 0;
    TopoDFS(node, visited, endTime, ref time);

    return endTime.OrderByDescending(x => x.Value).Select(x => x.Key);
}
```

## 第七章 查找

### 7.1 查找的基本概念

- 查找表：用于查找的数据集合
- 静态查找表：查找表的操作只涉及查找，不能修改
- 平均查找长度 ASL：

$$
ASL=\sum_{any\ item\ i}PossibilityOfFinding(i)\times TimesOfCompareToFind(i)
$$

### 7.2.2 折半查找

- 折半查找的判定树是一个 **平衡二叉树**
- 对于n个元素的顺序表，其对应的树的高度为 $ceil(\log_2(n+1))$

### 7.2.3 分块查找

- 对于分块查找，如果索引表和块内都用顺序查找，ASL：

$$
ASL=L_1+L_2=\frac{b+1}{2}+\frac{s+1}{2}
$$

其中，b是索引表的长度（假设均匀分块），s是每一块的长度：

$$
ASL=\frac{s^2+2s+total}{2s}
$$

如果 $s=\sqrt {total}$ 则ASL 最小，为 $\sqrt{total}+1$

```c
    new_root->left = root;
```

### 7.4.1 B树

- m阶B树当中：
    - 每个节点最多有m个子树，这也就意味着，最多有m-1个关键字
    - 对于根节点，如果他不是叶节点，则至少有两颗子树
    - 所有除了根节点的非叶节点至少有 $ceil(\frac m2)-1$个**关键字**，也就是至少有 $ceil(\frac{m}{2})$ 个子树
    - 关键字个数
    
    $$
    ceil(\frac m2)-1 \le keys \le m-1
    $$
    
- 分析B树的高度：
    - 如果要分析高度最小，那么每一层的子树必须最多，即每层都有m个子树
    
    $$
    n \le (m-1)(1+m+m^2+\dots+m^{(h-1)})=m^h-1
    $$
    
    其中：m-1是每一层最多的关键字个数，所以可以得出：
    
    $$
    h\ge \log_m(n + 1)
    $$
    
    - 如果要分析高度最大，那么每一层的子树必须尽可能地少，即 $ceil(\frac m2)$
    - 数到最后一层之后，叶节点个数表达式是 $2(ceil(\frac m2))^{h-1}$，而叶节点数量应当是 $n+1$
    - 所以有：
    
    $$
    n+1\ge2(ceil(\frac m2))^{h-1}\\
    h\le \log_{ceil(\frac m2)}(\frac{n+1}{2})+1
    $$

## 第八章 排序

- 对于n个关键字的基于比较的排序，其至少要进行 $ceil(\log_2(n!))$ 次比较
- 直接插入排序算法最好情况下（如果本来就有序）时间复杂度是 $O(n)$，最坏是 $O(n^2)$

### 8.3.1 冒泡排序

- 在最坏的情况下，冒泡排序需要 $(n-1) + (n-2) + \dots + 1 = \frac{n(n-1)}{2}$
- 王道上面，把一次交换看成是3次数据的移动（temp, a, b之间的相互赋值），如果每一次比较都要进行交换，那么最差情况下的移动次数就是 $3\times\frac{n(n-1)}{2}$

### 8.3.2 快速排序

- 快速排序理想的情况下时间复杂度是 $O(\log_2(n))$ 而在最坏了情况下（本来就已经有序了是 $O(n^2)$）
- 快速排序不是稳定的排序算法

### 8.5.1 归并排序

- 归并排序是一种稳定的排序算法

### 8.6.1 内部排序算法的比较

- 插入排序、冒泡排序、归并排序、基数排序都是稳定的排序算法
- 简单选择排序、快速排序、希尔排序和堆排序都是不稳定的排序算法

### 8.7.2 外部排序的方法

- 外部排序的总时间=内部排序的时间+外存信息读/写的时间+内部归并的时间
- 生成初始归并段的方法（置换-选择算法）

```csharp
public IEnumerator<List<int>>GetNextSegment(IEnumerator<int> file) {
		// 内存工作区
    int?[] workSpace = [null, null, null];

    for (int i = 0; i < 3; i++) {
        if (file.MoveNext()) {
	          // 先取出3个放入工作区
            workSpace[i] = file.Current;
        }
    }
	
		// 只要内容不是全空，说明还有新的段可以生成
    while (workSpace.Any(x => x is not null)) {
        List<int> segment = [];
				
				
				// 选出所有的比段中末尾的值更大的数 numbers
        var query = from i in Enumerable.Range(0, 3)
                    where workSpace[i] is not null && 
                        (segment.Count == 0 || workSpace[i] > segment[^1])
                    select (i, workSpace[i]);
				
				// 只要有这样的数
        while (query.Any()) {
		        // 取其中最小的那一个，加入到工作区
            int index = query.MinBy(x => x.Item2).Item1;
            segment.Add(workSpace[index].Value);
            workSpace[index] = null;
						
						// 尝试用新的数补充
            if (file.MoveNext()) {
                workSpace[index] = file.Current;
            } else {
                workSpace[index] = null;
            }
        }

        yield return segment;
    }

    yield break;
}

List<int> input = [17, 21, 05, 44, 10, 12, 56, 32, 29];

var enumerator = input.GetEnumerator();
var segments = GetNextSegment(enumerator);

while (segments.MoveNext()) {
    Console.WriteLine(string.Join(", ", segments.Current));
}
```
