# 计组缩写名词/概念

## 第一章

- MAR：Memory Address Register 存储器地址寄存器
- MDR：Memory Data Register 存储器数据寄存器
- PSW：Program Status Word 程序状态寄存器
- ALU：Arithmetic and Logic Unit 算数逻辑单元
- ISA：Instruction Set Architechture 指令集体系结构，定义了一台计算机可以执行的所有指令的集合，每条指令规定了计算机执行什么操作，以及所处理的操作数存放的地址空间和操作数类型。
- 机器字长（简称字长）：计算机进行一次整数运算所能处理的二进制数据的位数
- 指令字长：一条指令的二进制长度
- 存储字长：一个存储单元的二进制长度
- 存储字：一个存储单元可以存好几个二进制代码，一个存储单元里包含的二进制数据就是存储字
- 数据通路带宽：（外部）数据总线一次所能并行传送信息的位数
- 吞吐量：系统在单位时间内处理请求的数量
- 响应时间：发送一个请求到系统作出响应并获得结果需要的等待时间
- 时钟脉冲信号：由机器的脉冲源发出的脉冲信号经整形和分频后形成
- 主频：机器内部主时钟的频率，时钟周期的倒数
- CPI：Cycle Per Instruction：执行一条指令需要的时钟周期数量
- IPS：Instruction Per Second：每秒执行多少条指令
- MIPS：Million Instruction Per Second
- FLOPS：Floating-point Operations Per Second 每秒执行多少次浮点运算
- 计数单位前缀：（每两个之间相差$10^3$）
    - M-Million百万
    - G-Giga十亿
    - T-Tera万亿
    - P-Peta千万亿
    - E-Exa百京
    - Z-Zetta十万京

- 系列机：具有基本相同的体系结构，使用相同基本指令系统的多个不同型号的计算机组成的一个产品系列
- 兼容：运行在某个型号的计算机系统中的软硬件也能在另一个型号的计算机系统上运行
- 固件：将程序固化在ROM中组成的部件，是一种具有软件特性的硬件

背诵型概念解释：

- 冯诺依曼机的特点：

  - 存储程序
  - 指令和数据同等地位
  - 五大设备：**运算器、控制器、存储器、输入设备、输出设备**
  - 二进制

- 冯诺依曼机的基本控制方式：控制流驱动方式

- 现代计算机以 **存储器** 为核心

- CPU区分指令和数据的依据是 **指令周期的不同阶段**

- PC 不是透明的

## 第二章

- 基数：每个数位用到的不同数码的个数
- 位权：数码表示的数值和自己数码本身的字面值的比值（例如100中1的位权是100）
- 真值：带有正负符号的值
- 机器数：把符号和数值部分一起编码，完全数字化的数
- BCD码：把十进制的每一位都用四个二进制来表示，各自的位权是8 4 2 1（8421编码）
- 大端：从最高有效字节到最低有效字节（符合人类直觉的）0xabcd -> 0xab|0xcd
- 小端：和大端相反 0xabcd -> 0xcd|0xab

**在IEEE浮点数中，阶码的位移量是 $2^{n-1}-1$ 而不是通常的 $2^{n-1}$**

## 第三章

RAM：Random Access Memory 随机存储器
ROM：Read-Only Memory 只读存储器
串行访问存储器：在读写操作时会根据其物理位置进行先后寻址
直接存储存储器：介于RAM和串行访问存储器之间，在大范围内随机访问，在小块内顺序查找

存储元：存放一个二进制位的物理器件

存取时间：启动存储器操作到完成该操作需要的时间
存取周期：进行一次完整的读写操作需要的时间，包含存取时间和复原时间
主存带宽：每秒从主存进出信息的最大数量

相联存储器：内容指定方式和地址指定方式相结合的存储器

SRAM：Static RAM 静态随机存储器，使用触发器之类的存储信息，很快但贵，非破坏性读出
DRAM：Dynamic RAM 动态随机存储器，使用电容电荷存储信息，慢，功耗小，需要定时刷新

集中刷新：一个刷新周期内给一个固定时间把所有的存储器都刷新
分散刷新：工作周期分成两部分，一部分刷新，一部分进行读写
异步刷新：刷新周期中每一行只刷新一次，可以认为刷新定时是独立于读写的一个单独的过程

突发传输：给出一个首地址，然后传输连续的一片数据（可能更准确的翻译是批量传输）

SDRAM：Synchronized Dynamic RAM 同步DRAM，其读出写入和CPU的频率同步，需要的时钟周期是已知的，CPU无需反复采样等待

MROM：Maskable ROM 掩模式只读存储器：在出厂的时候内容就直接写好，之后无法改
PROM：Programmable ROM 一次可编程只读存储器：用户可以用编程器写进程序，写了之后就无法改
EPROM：Erasable Programmable ROM 可擦除可编程只读存储器：可以写也可以改 **不能取代RAM**，因为编程次数有限，而且写入时间过长
Flash 存储器：在EPROM的基础上，兼顾ROM和RAM，偏移，擦除重写快，集成度高
SSD：Solid State Drive 固态硬盘：Flash和控制单元组成，长期保存信息，快速擦除重写，价格高

MM：Main Memory 主存储器（内存）

多模块存储器：一种空间并行技术，用多个结构相同的存储模块并行工作来提高存储器的吞吐率，常用的有单体多字存储器和多体低位交叉存储器

单体多字存储器：每个存储单元存储m个字，总线宽度也是m个字，一次并行读出m个字，连续存放的时候存取快
多体并行存储器：
- 高位交叉编址：一个存储器占用一片连续的空间，这片空间之后到下一个存储器（不是并行）
- 低位交叉编址：一个存储器只占用碎片化的空间，并行。
  - 如果要实现轮流启动，设存取周期为T，总线周期为r，那么模块数量至少要大于等于T/r

理想情况下，m体交叉存储器可以每隔1/m存储周期读写一个数据

片选信号产生方式：线选法，译码片选法

磁盘驱动器：驱动磁盘转动并且在盘面上通过磁头进行读写的装置
磁盘控制器：磁盘驱动器与主机的接口，接受和解释CPU的平陵，发送控制信号和监测磁盘驱动器状态

磁头数：记录面数
柱面数：一个磁盘每个盘面上有多少个磁道
扇区数：每个磁道上有多少个扇形存储区域

磁盘高速缓存 Disk Cache：内存中开辟一片区域缓冲被送到磁盘上的数据

RAID：Redundant Arrays of Independent Disks 独立冗余磁盘阵列

- RAID0：无冗余无校验
- RAID1：镜像磁盘阵列
- RAID2：使用纠错海明码
- RAID3：位交叉奇偶校验
- RAID4：块交叉奇偶校验
- RAID5：无独立校验的奇偶校验

RAID0：数据块交替放在不同物理磁盘扇区中并行读取（条带化技术）

磨损均衡
- 动态磨损均衡：优先选择擦除次数少的新闪存块来写入
- 静态磨损均衡（更先进）：监测并自动进行数据分配，让老的闪存块以读为主，新的闪存块腾出空间来承担写的工作

读Cache相关操作（包括不命中处理，Cache替换）都是由硬件实现的

全写法：同时写Cache和内存
回写法：只写Cache，加一个脏位，Cache对应内容失效时写入到内存（有数据不一致的危险）

写，Cache不命中：
写分配法：更新主存单元然后调入Cache
非写分配法：不调入Cache，只写主存单元

## 第四章

指令：指示计算机执行某种操作的命令，一条指令对应一条汇编语言

隐含寻址：不明显给出操作数地址，比如累加 ACC

变址寻址：有一个专门的变址寄存器 IX，地址为 (IX) + A

调用者保存寄存器：P调用Q时，这些寄存器前后的值可能因为Q的修改不一样
（EAX ECX EDX）
被调用者保存寄存器：可以断言调用Q的前后这些寄存器的值是一致的
（EBX ESI EDI）

CISC：Complex Instruction System Computer 复杂指令系统计算机 指令系统庞大，长度不固定，访存指令不受限制，使用频度相差大，有的指令可以非常复杂以至于无法用硬件优化
RISC：Reduced Instruction System Computer 精简指令系统计算机 指令系统简单，只有一些频度最高的简单指令，CPU通用寄存器数量多，一定使用指令流水线技术，大部分指令在一个时钟周期内完成

VLSI：Very Large Scale Integration circuit 超大规模集成电路

## 第五章

GPR：General Purpose Register 通用寄存器组
PSW：Program Status Word 程序状态字
SR：Shift Register 移位寄存器

CPI：Clock Per Instruction 每指令需要的时钟周期数

组合逻辑单元：输出只取决于当前的输入（不包含存储）
时序逻辑单元：输出取决于当前输入和之前的输入（必然包含存储），必须在时钟节拍下工作

数据通路：数据在指令执行过程中所经过的路径，包括路径上的部件

微命令：控制部件向各执行部件发出的各种控制命令
微操作：各个控制部件收到微命令后执行的操作

CM：Control Memory 控制存储器 用于存放微程序（ROM）

异常：由CPU内部产生的意外事件（内中断）
中断：CPU外部发送的中断请求（外中断）

故障（Fault）：指令启动后，执行结束前的异常事件
自陷（Trap）：利用事先设定好的指令或者特殊标志让它在被CPU执行完成后进行相应的处理
终止（Abort）：执行指令的过程中发生了计算机无法继续执行的硬件故障，程序只能终止

INTR：Interruption 可屏蔽中断请求线
NMI：Non Maskable Interruption 不可屏蔽中断请求线

软件识别方式：让操作系统根据CPU的异常状态寄存器来进行中断处理
硬件识别方式：所有中断向量放在一个中断向量表中，根据类型号快速找到处理程序

空间上的并行：超标量处理机（一个处理机内设置多个执行相同任务的功能部件）

结构冒险：不同指令同时争用同一个部件
数据冒险：后面的指令要用的数据依赖于前面还未产生或未写入到存储的结果

超标量流水线技术：动态多发射技术，每个时钟周期内并发多条独立指令

超长指令字技术：又称静态多发射技术，由编译程序挖掘出指令间嵌在的并行性，将多条能并行操作的指令组合成一条具有多个操作码字段的超长指令字

超流水线技术：提高流水线主频

细粒度多线程：不同线程之间的指令轮流交叉执行
粗粒度多线程：一个线程在遇到较大开销的阻塞的时候切换，这种多线程的线程切换的开销比细粒度多线程大
SMT：Simultaneous Multi Threading 同时多线程，指令集并行，一个时钟周期内执行多个指令

SMP：Shared Memory Processer 具有共享的单一物理地址空间的多处理器

UMA：Unified Memory Access 统一存储访问，每个存储器对所有存储单元的访问时间大致相同
NUMA：Non-Unified Memory Access 非统一存储访问，某些存储器的访存速度比其他的要快

QPI总线：Intel QuickPath Interconnect CPU之间使用QPI总线相互连接，使用QPI访问其他CPU的远程内存（存在于NUMA中）

## 第六章

片内总线：芯片内部的总线
系统总线：计算机系统内各功能部件之间相互连接的总线

IO总线：连接IO设备
通信总线：计算机系统之间或者计算机与其他系统之间传送信息的总线

同步总线：总线连接的部件或者设备有统一的时钟进行同步
异步总线：总线连接的部件或者设备没有时钟进行同步，而是使用信号握手的方式来进行协调，操作时序是不固定的

双总线：主存总线、IO总线

三总线：主存总线、IO总线、DMA总线

总线的作用：传输**数据**，**地址**，**控制信息**

总线复用：一种信号线在不同的时间传输不同的信息

同步定时方式：系统采用统一的时钟信号协调发送和接受双方的传送定时关系
异步定时方式：没有统一的时钟，没有固定的时间间隔，完全依靠传送双方相互制约的握手信号来实现定时控制
半同步定时方式：使用一个Wait信号线来标志数据有没有准备完成，使用轮询采样的方式
分离式定时方式：将总线事务分解为请求和应答两个子过程，每次都进行请求

## 第七章

独占查询：一旦设备启动了CPU就全精力投入到IO当中不停轮询，占用100%
定时查询：设备启动了之后CPU只是周期性地定时查询一次

断点：中断发生时，CPU在回来之后预期从哪里继续执行程序（PC的值），断点的保存由硬件实现
现场：中断发生时CPU内部的工作状态如寄存器内容，程序状态字，堆栈指针等等，现场的保存由软件（中断程序）实现

INT：Interruption 中断标记触发器
EINT：Enable Interruption 中断允许触发器

中断响应优先级：由判优电路来决定
中断处理优先级：由中断屏蔽来决定

HRQ：Hold Request 请求保持，当HRQ=1，表示DMA请求CPU暂时让出总线
HLDA：Hold Acknowledge 保持响应，当HLDA=1，表示CPU已经让出总线（即接受HRQ）
