    - 运算器（数据处理）
    - 存储器（数据处理辅助）

    - 输入设备（数据来源）
    - 输出设备（数据去向）

⚠️⚠️⚠️内存就已经是主存了，寄存器是比主存更牛逼的存在
- MAR（Memory Address Register）存储器地址寄存器和MDR（Memory Data Register）存储器数据寄存器都是主存储器的组成，但是也同时在CPU里面（物理上寄存器应该是在CPU里面）

    - 存储器地址存储器（订单）

    - 时序逻辑（管理员）

- 高级语言源程序转化为可执行目标文件主要过程：
预处理—>编译—>汇编—>链接

- 计算机的主要性能指标有：
    - 机器字长
    - 数据通路带宽（数据总线一次能并行传送信息的位数）
    - 主存容量（MAR的位数反映了存储单元的位数（不要在这里考虑虚拟内存），MDR的位数反映了存储单元的字长）
    - 运算速度
        - 吞吐量：单位时间内处理请求的数量
        - 响应时间：用户向系统发送请求后到系统做出响应并获得结果的等待时间

    - PFLOPS: $10^{15}$（Peta）
    - EFLOPS: $10^{18}$（Exa）
    - ZFLOPS: $10^{21}$（Zetta）

## 第二章 数据的表示和运算

- 现代计算机一般用补码表示整数，用原码小数表示浮点数的尾数部分，用移码表示浮点数的阶码部分。
- 对于 $n+1$ 位的原码，其表示范围是 $[-(2^{n}-1),2^n-1]$，原码的一大好处是乘除运算比较简便（但是0会有正负0的两个表示）
- 对于 $n+1$ 位的补码，其表示范围是 $[-2^n,2^n-1]$
- 对于 $n+1$ 位的移码，偏置值一般取 $2^n$，则其表示范围是： $[-2^n,2^n-1]$，移码常用来表示浮点数的阶码
- 对于反码和补码的负数，其数值部分的直接翻译值越小，则绝对值越大，负的越多。
- 变形补码，有双符号位。00表示正，11表示负（左符是真正的符号位，右符用于判断是否溢出）

注意！！！！！

C语言规定如果有符号数和无符号数同时发生运算，按照无符号数计算

加法器：

- 溢出标志OF Overflow Flag $OF=C_n \oplus C_{n-1}$ 用于判断 **有符号数** 的运算是不是溢出了
- 进位借位标志CF Carry Flag $CF=C_{out} \oplus C_{in}$ 用于判断 **无符号数** 的运算是不是溢出了

- 无符号数大小的比较：
    - ZF=1：作差为零，A=B
    - ZF=0, CF=0：有富余，A>B
    - ZF=0, CF=1：不够减，A<B
- 有符号数大小的比较：
    - ZF=1：作差为零，A=B
    - OF=SF，ZF=0：要么是溢出了后为负数，说明A-B正向溢出，A>B，要么是没有溢出为正数说明有富余，A>B
    - OF≠SF，ZF=0：要么是溢出了之后为正数，说明A-B负向溢出，A<B，要么是没有溢出为负数，说明不够减，A<B
- 乘法的递推： $P_i=2^{-1}(P+X\times y_i)$   $P_n$是最后要求的结果
- 小端和大端：

```csharp
// 现代计算机一般是小端，所以可以观察到实际上的输出格式是：0x78 0x56 0x34 0x12
var isLittleEndian = BitConverter.IsLittleEndian;
System.Console.WriteLine($"Current machine is little endian: {isLittleEndian}");
unsafe
{
    int value = 0x12345678;
    byte* ptr = (byte*)&value;

    Console.WriteLine($"Actual value (as hex): 0x{value:X8}");
    for (int @byte = 0; @byte < sizeof(int); @byte++)
    {
        Console.WriteLine($"Byte {@byte}: 0x{ptr[@byte]:X2}");
    }

    System.Console.WriteLine();

    System.Console.WriteLine("Big endian schema:");
    for (int @byte = sizeof(int) - 1; @byte >= 0; @byte--)
    {
        Console.Write($"|0x{ptr[@byte]:X2}");
    }
    System.Console.WriteLine("|");

    System.Console.WriteLine();

    System.Console.WriteLine("Little endian schema:");
    for (int @byte = 0; @byte < sizeof(int); @byte++)
    {
        Console.Write($"|0x{ptr[@byte]:X2}");
    }
    System.Console.WriteLine("|");
}
```

**浮点数的表示**

- 在IEEE754当中，浮点数当中的阶码（使用移码表示）使用的偏置值是 $2^{n-1}-1$ 而不是 $2^{n-1}$
- float 由于阶码全1，全0时有特殊含义，因此阶码的范围实际上是： $[1-127, 2^{8}-2-127]$
- 阶码全一：
    - 如果尾数不为0，则表示NaN
    - 如果尾数为0，则表示无穷
- 阶码全0：
    - 如果尾数为0，表示两个方向上的数值0
    - 如果尾数不为0，表示非规格化浮点数
- float：偏置值取127
- double：偏置值取1023
- 浮点运算只有在 **阶码的表示能力不足时** 才发生溢出，和尾数没什么关系
- 浮点数的加法运算过程中，是小阶码向大阶码看齐（即使用右规，使小阶码增大），并且这个过程中，移出的数据暂时不会丢掉，来优化之后的计算精度

## 第三章 存储系统

- 存储系统的三个性能指标：
    - 存储容量
    - 单位成本
    - 存储速度：
        - 存取时间（ $T_a$ ）
        - 存取周期（ $T_m$ ）
        - 主存带宽（ $B_m$ ）
        
        **值得注意的是，存取周期并不等于存取时间，**存取周期不仅仅包括存取时间（执行这个操作），并且包括在存取后进行恢复的时间（复原时间），即等待一段时间准备下一次存取操作。
    
- 主存和 Cache 之间的操作是由硬件完成的，所有程序员（**包括系统程序员**）对这件事情都是 **认知透明的**
- 主存和辅存之间的数据调动需要涉及操作系统的调度，只是 **对应用程序员透明**
- **SRAM和DRAM**
    - SRAM: Static Random Access Memory：静态随机存储器，使用触发器等实现，不需要再生，速度快，功耗大，价格高。（Cache等）
    - DRAM: Dynamic Random Access Memory：动态随机存储器，使用晶体管/电容实现，需要再生，速度慢，功耗小，价格低。（内存等）
- DRAM的刷新：
    - 集中刷新：在一个刷新周期内，一段固定的时间对所有行刷新一次
    - 分散刷新：工作周期前一部分用于读写，后一部分用于刷新：（读取和刷新是同步的，按书上的理解，基本就是读写完后就刷新一下，但是刷新地没有集中刷新那么多，可能就刷新一行或者几行）
    - 异步刷新：一个刷新周期内每行刷新一次（异步刷新是由系统外部或控制器根据自己的时钟和策略手动发起的刷新，不跟随内存的读写周期。刷新时 DRAM 被锁定，读写暂停。）
- **引脚复用技术** 指的是行和列坐标先后串行输入进去
- 引脚数量优化的过程当中，尽量让 **行和列的数量差不多**，在这个前提下，**尽可能地让行更少一点**（因为刷新是按行刷新的，行少一点能减少刷新开销）
- 目前更多地采用 **SDRAM （Synchronized DRAM）** 芯片，使得芯片按照CPU-内存总线的最高速度运行，让CPU基本上不用去等待 RAM 操作，而是在 RAM 操作的过程当中去干点别的。
- MDR 的位数永远和 **数据线的位数** 相同，即使数据线位数不等于存储字长（*MDR 只承担存储单元的角色，不去承担组装数据的职责*）
- 对于交叉存储器（交叉方式编址），如果要达到性能最大：
    - 设存取周期是 $T$，总线周期是 $r$，则 $m=\frac Tr$
    - 按每隔 $\frac 1m$的时间来启动一个模块，那么模块的数量必须要 $\ge m$，这样才能使得存取的流水线不中断（存取启动完最后一个模块后，等待 $\frac{1}{m}$ 时间就能再去启动第一个模块）
    - 采用这种方案，在流水线填满后，只需要等待一个总线周期就能立即存取下一个字，所以连续存取 $m$ 个字的时间为： $T+(m-1)r$
    - 而传统方案（顺序方案）的时间为： $mT$
    - 但是如果是随机存取，则可能出现访存冲突，那么此时必须暂停
- 在计算交叉随机仿存序列的时候注意：
    - 比如A B B A，其中B和B的访存出现冲突，导致B和B之间有一段延迟时间
    - **而这段延迟时间恰好顶开了两次A的仿存，使得两次A之间不会冲突**
    - A B (wait) (wait) B A（假设访存周期是3个单位）

**外部存储器**

- 在SSD当中，读写的单位是“页”，而擦除的单位是块
- 一页所属的块整个被擦除之后，才能写一次。
- 擦除整个块后，块中的每个页都可以直接再写一次
- 固态硬盘的随机写很慢，因为擦除很慢。如果试图操作一个已经包含数据的页，则必须把它里面的数据都复制到一个新的块中，才能进行写操作。
- 为了增加SSD的实际使用寿命，使用了：**动态磨损均衡和静态磨损均衡** 来平均闪存块的磨损。

**Cache**

- Cache由SRAM组成，通常直接集成在CPU当中
- Cache映射的三种方式：
    - 直接映射：主存内容和Cache位置一一对应不可变化
        - 格式：Tag CacheRow Offset
    - 全相联：主存内容可以在Cache中任意一个位置
        - 格式：Tag Offset
    - 组相联：主存内容和Cache组一一对应不可变化，组内可以存在任意位置（前两种的折中）
        - 格式：Tag Set Offset
        - 如果只有一组，那么就是全相联
        - 如果每组只有一个行，那么就是直接映射
- Cache的r路组相连映射指的是 **一组有r行CacheItem**
- 组相连的一个模拟：

```csharp
public class GroupedCache
{
    private CacheItem[,] _cache;

    public GroupedCache(
        int rows,
        int groupSize,
        int cacheBlockSize
    ) {
        _cache = new CacheItem[rows, groupSize];
        for (int i = 0; i < rows; i++)
        {
            for (int j = 0; j < groupSize; j++)
            {
                _cache[i, j] = new CacheItem()
                {
                    Tag = null,
                    Data = new byte[cacheBlockSize]
                };
            }
        }
    }

    public byte? Read(int tag, int set, int offset)
    {
        for (int i = 0; i < _cache.GetLength(1); i++)
        {
            if (_cache[set, i].Tag == tag)
            {
			          // 命中
                return _cache[set, i].Data[offset];
            }
        }
        // 未命中
        return null;
    }
}

public struct CacheItem
{
    public int? Tag { get; set; }
    public byte[] Data { get; set; }
}
```

- 对比：

| **性能** | **全相联** | **直接映射** |
| --- | --- | --- |
| 命中率 | ✅高 | ❌低 |
| 判断开销 | ❌高 | ✅低 |
| 额外空间占用 | ❌大 | ✅小 |

而组相联是上述两者的折中

- 对于非直接映射的 Cache，需要考虑替换算法：
    - FIFO
    - RAND
    - LRU（Least Recently Used）最近最少使用
    - LFT （Least Frequently Used）最不经常使用
- 在访问的存储区超过 Cache 组的大小的时候，命中率会变得很低，这种现象叫做 **抖动**
- 为了保持缓存一致有两种方法：
    - 全写法：同时写缓存和主存（可以加一个写缓冲来缓和速度不一致）
    - 回写法：只写缓存并设置脏位，在Cache替换时写回主存（易导致数据不一致，尤其是多核心CPU的情况下，这就是volatile关键字的作用）
- 如果Cache的写不命中，也有两种方法：
    - 写分配法：（把主存块放进Cache写）（通常配合回写法）
    - 非写分配法：只更新主存，不调进Cache（通常配合全写法）

**虚拟存储**

- 虚拟存储时对实际存储的抽象，屏蔽了底层物理地址的复杂性，隔离了不同进程之间的内存，并且扩大了存储空间
- 虚拟存储把主存当做辅存的 **Cache**，主存里存不下的部分存到辅存上面去

**页式虚拟存储器**

- 页表每一项的结构：
    - 有效位（这一项是不是有效的，一位）
    - 脏位（这一项是不是被修改了，一位）
    - 引用位（标记这一项是不是被使用，一位，用于配合LRU等）
- **页表本身也可能会被置换出去**，但是注意，每个进程会有一小部分元数据常驻内存，这样当页表换出去的时候，就有方法找到页表在磁盘上哪个位置
- 缺页处理是要有操作系统介入的，因为要使用操作系统的缺页异常处理程序
- 一级页表的一个模拟：

```csharp
public class PageMemoryTranslator
{
    private List<PhysicalPageItem> _physicalPages;

    public void Translate(int virtualPage, out int physicalPage)
    {
        var item = _physicalPages[virtualPage];
        if (item.Active)
        {
            physicalPage = item.PhysicalMemory;
        }
        
        // 缺页处理（模拟）
        physicalPage = LoadPageFromDisk(virtualPage);
        _physicalPages[virtualPage] = new PhysicalPageItem { Active = true, PhysicalMemory = physicalPage };
    }

    private int LoadPageFromDisk(int virtualPage)
    {
		    // 这里实际上是从磁盘读入到主存，可能还涉及已有页面的替换。
        return Random.Shared.Next(0, 100);
    }
}

public struct PhysicalPageItem
{
    public PhysicalPageItem()
    {
        Active = false;
        PhysicalMemory = 0;
    }

    public bool Active { get; set; }
    public int PhysicalMemory { get; set; }
}
```

- TLB的确实既可以用软件来处理，也可以用硬件来处理
- 段式虚拟存储器的优点是段的分界与程序的自然分界对应，具有逻辑独立性，易于编译，管理，修改，保护，便于多道程序的共享
- 段式虚拟存储器的一个缺点是：
    - 段的长度是 **可变的**，导致出现较多的段间内存碎片

- 段页式虚拟存储器是段式虚拟存储器和页式虚拟存储器的折中，程序按逻辑结构分段，每段划分为固定大小的页。
    - 优点是其兼顾了段式和页式存储器的优点，按段实现共享和保护
    - 缺点是其地址变换的过程中要查两次表
- CPU和Cache和主存之间都有直接通路，但是CPU和辅存之间没有直接通路
- 主存内部的数据调入Cache和CPU可以是同时的，但是辅存里面的数据要进CPU只能先放进主存，然后再重试访问

## 第四章 指令系统

- ISA规定的主要内容包括：
    - 指令的格式，指令寻址方式，操作类型，以及每种操作对应的操作数
    - 操作数的类型，操作数寻址方式，按大端还是小端
    - 程序可以访问的寄存器编号，个数，和位数，存储空间大小和编址方式
    - 指令执行过程的控制方式，程序计数器，条件码定义等
- 指令字长和机器字长没有必然的关系
- 指令的寻址方式
    - 指令寻址
        - 顺序寻址：PC+1加载指令
        - 跳跃寻址：跳转指令
    - 数据寻址
        - 立即寻址：指令中的字段就是操作数（立即数）
        - 直接寻址：指令中的字段是内存中的地址
        - 寄存器寻址：指令中的字段标识了数据在哪个寄存器
        - 寄存器间接寻址：指令中的字段标识了数据的地址在哪个寄存器
- CISC和RISC
    - CISC： Complex Instruction System Computer 复杂指令系统计算机：
        - 指令系统非常庞大
        - 指令的长度不固定，格式多，寻址方式多
        - 可以访存的指令不受限制
        - 各种指令的使用频度相差很大
        - 大多使用微程序控制，有些指令太复杂以至于无法用硬连线的方式来控制
    - RISC：Reduced Instruction System Computer 精简指令系统计算机
        - 只选取一些比较简单的指令，复杂指令用简单指令进行组合
        - 一定使用指令流水线技术，大多数的指令在一个时钟周期内就能完成
        - 硬布线控制为主，少用或者不用微程序控制
        - 只有LOAD和STORE去访问主存，其他的指令操作都在寄存器之间进行
        - 有很多的通用寄存器

## 第五章 中央处理器

- CPU的基本结构有：运算器（执行者）和控制器（导演）
- 指令寄存器（IR  Instruction Register）的位数等于指令的字长
- 现在的计算机当中，认为时钟周期就是 **节拍，**而在每一个节拍当中，CPU要控制具体器械的操作，这些操作需要发出一个一个更细的信号，比如：读内存某某某，寄存器某某某打开写入通道，这些信号就是 **工作脉冲**
- CPU 在 **执行周期** 结束之后会查看是否有中断请求
- 一个指令的周期：
    - 取指周期
    - 间址周期
    - 执行周期
    - 中断周期
- 处理器的分类：
    - 单周期处理器：所有的指令都要用相同的执行时间来完成，即使有的指令时间很短，但是也必须拉长
    - 多周期处理器：一个指令要几个周期就给几个周期而不是强制拉长成一样的时间
    - 流水线处理器：不同指令先后进入流水线一起执行，理想情况下每个时钟周期可以完成一条指令
- 组成数据通路的元件主要分为组合逻辑元件和时序逻辑元件两类
    - 组合逻辑元件（控制元件）：任何时刻的输出都之和那一时刻的输入有关
    - 时序逻辑元件（状态元件）：任何时刻的输出不仅仅和当时的输入有关，还和那一时刻之前的输入有关
- 数据通路基本结构：
    - CPU内部单总线：所有的元件都连到一个总线上，冲突的概率很大
    - CPU内部多总线：元件分类连到不同的总线上，冲突概率降低
    - 专用数据通路：直接按照数据流动方向来安排电路，不会冲突，性能好，但是硬件量大
- ALU中要设置一个寄存器，因为其两个输入可能都来自于总线，而总线同一时间只能给出一个值，所以第一个值来了之后，必须先存到某个地方
- 控制器一般有两种实现方案：
    - 硬布线控制器：指令内部先后控制信号的发出由电路线直接控制
    - 微程序控制器：采用一个专门的微程序来解释执行指令
- 微命令之间有相容性和互斥性，有的微命令可以同时发出而有的微命令不能同时发出
- 一条微命令至少有两个部分：
    - 操作控制字段，用于产生操作要发出哪些信号
    - 顺序控制字段，用于控制产生下一条要执行的微指令的地址
- 微周期是从控制存储器中取出并执行一条微命令需要的时间，通常微一个时钟周期
- 微指令的编码有三种：
    - 直接编码：每一位代表一个控制信号要不要发生
    - 字段直接编码：把互斥的放在一块，译码后才是控制信号
    - 字段间接编码：一个字段的某些微命令要根据另一个字段的某些微命令来解释，有些字段的含义是不确定的，必须根据另一个字段来
- 水平型微指令直接对应控制信号，有输出的时候是1，没有输出的是0，便于让多个行为并行进行
- 垂直型微指令设置微操作码，目的地址和源地址，并行性更差
- 微程序控制器相比于硬布线控制器更容易修改

**异常和中断机制**

- **异常** 是CPU内部产生的，而 **中断** 是外来因素产生的
- 对异常和中断的处理是要有 **操作系统** 来完成的
- 异常的分类：
    - 故障（Fault）是在指令启动后，执行结束之前监测到的异常事件，一些故障可以恢复
    - 自陷（Trap）使用一些特殊指令，主动地进行可以预料到的异常来实现某些功能
    - 终止（Abort）在执行指令的过程当中出现的让计算机无法继续执行的硬件故障。**这是随机发生的，而前两者都与具体的指令有关**。严重问题，直接导致程序崩溃死亡。
- 中断的分类：
    - 可屏蔽中断，通过可屏蔽中断请求线INTR发来的中断请求
    - 不可屏蔽中断，通过NMI（Non-Maskable Interrupt）发来的中断请求，通常非常紧急，不能忽略屏蔽
- 通过调整是否允许中断IF（Interrupt Flag）来实现是否响应中断
- 两种识别方式：
    - 软件识别方式，CPU设置一个异常状态寄存器，操作系统根据他来处理
    - 硬件识别方式，向量中断，在中断的时候直接根据类型号找到处理程序

**指令流水线**

- 在空间上进行并行，即在一个处理机内设置多个执行相同任务的功能部件一起工作，叫做 **超标量处理机**
- 指令流水段的个数以 **最复杂的指令多用的功能段数为准**
- 流水段的长度以 **最复杂的操作所用的时间为准**
- 流水线只有在大量任务输入的时候才能表现出性能提升，对于 **零散，单个的任务** ，其耗时反而会延长
- 流水线有三种冒险：
    - **结构冒险**，不同指令在同一时间都想用同一个硬件资源（比如都想读写主存，而主存的读写通道没有分离）
    - **数据冒险**，不同指令先后想要操纵同一个数据，但是后来的指令要依赖前一条指令的结果，导致可能前一条指令还没将答案写入数据，后一条指令就拿着老数据开始执行了（RAW Read After Write）
    - **控制冒险**，遇到分支跳转指令，或者有中断异常的时候，程序执行位置会突然改变，导致流水线断流
- 解决数据冒险有两种方法：
    - 使用nop进行延迟处理
    - 使用旁路转发
- load-use数据冒险无法使用转发进行处理，因为数据根本还没有读出来，也就没有转发一说。最好的方法是 **在编译期进行优化**，尽量少出现这种问题，实在出现了，使用nop进行延迟处理
- 控制冒险使用nop指令进行延迟，而现代CPU里有分支预测，乱序执行等方案进行处理
- 按理想情况来算，一个处理器从非流水线变成使用流水线，使用流水线的加速提升比率是k，其中k是流水线的段数
- 高级流水线技术有：
    - 超标量流水线技术：一个时钟周期并发多条独立指令
    - 超长指令字技术：把多条能够并行操作的指令组合成一个有多个操作码字段的超长指令字
    - 超流水线技术：提高流水线的主频
- 理论上来说，流水线段数越多提速越好，但是用于流水寄存器的开销实际上也会更大，所以并不是越多越好
- 多发射流水线能够做到CPI<1，而超流水线还是CPI=1，但是主频更高
- 多处理器的基本概念：
    - SISD（Single Instruction Single Data）
    - SIMD（Single Instruction Multiple Data）（比如向量处理器）
    - MIMD（Multiple Instruction Multiple Data）
        - 分为多计算机系统和多处理器系统
        - 多计算机系统之间地址独立，只能通过消息传递共享数据，又称 *消息传递MIMD*
        - 多处理器系统之间地址共享，又称 *共享存储MIMD*
- 硬件多线程的概念：
    - 细粒度多线程：每个时钟周期都可以切换
    - 粗粒度多线程：连续好几个时钟周期都运行当前线程，只有遇到较大的阻塞的时候才切换
    - 同时多线程：真正意义上的在同一个时钟周期发射多个指令，Intel处理器中的超线程就是这个
- 单一地址空间的多处理器有两种类型：
    - UMA  （Unified Memory Access）统一存储访问多处理器
    - NUMA（Non-Unified Memory Access）非统一存储访问多处理器
- 区别在于，NUMA对于一个CPU，会有一个区域的访问比其他地方更快（本地内存和远程内存）

## 第六章 总线

- 总线的两个特点 **分时和共享**
- 总线设备：
    - 主设备：发出总线请求，能获得总线控制权
    - 从设备：只能响应主设备传来的命令
- 并行总线的速率并不是一定比串行总线高
- 系统总线的结构：
    - 单总线：所有的设备都共享同一条总线
    - 双总线：CPU到主存的总线和IO总线
    - 三总线：CPU到主存的总线，IO总线，DMA总线
- 总线的时钟周期和机器的时钟周期是同步的
- 传输周期：完成一次总线操作需要的时间
- 总线的工作频率：一秒钟能完成多少次数据传输。区别于总线的时钟频率，工作频率等于时钟频率/总线周期
- 总线事务的五个阶段：
    - 请求阶段：主设备请求总线的控制权
    - 仲裁阶段：总线仲裁机构决定下一个传输周期的总线控制权分配给哪个主设备
    - 寻址阶段：主设备把要访问的数据的地址与命令传送给从设备
    - 传输阶段：主设备和从设备进行数据交换
    - 释放阶段：主设备的相关信息都从总线上面撤除，让出总线的使用权
- 总线定时的方式：
    - 同步定时方式：主从设备之间有统一的时钟信号来同步传送定时关系（更快，但是不能及时进行数据的有效性检验）
    - 异步定时方式：依靠双方的握手信号来实现控制，可以适配两个工作速度相差很大的设备之间的协同工作（但是更慢，而且更复杂）
        - 不互锁方式：主设备请求之后不管从设备有没有发送回答，一段时间后都自动撤回
        - 半互锁方式：主设备请求之后必须要知道从设备发送回答了之后才撤回
        - 全互锁方式：不仅仅主设备要知道从设备发送回答了之后才撤回，从设备也要在知道了主设备撤回了之后才撤回
    - 半同步定时方式：设置一个 wait 线，接收方在时钟后沿时刻轮询一次wait，相比于异步来说更简单，而且也能适配不同速度设备之间的协同工作，但是系统时钟频率要求不能太高，所以总台来说也不是很快
    - 分离式定时方式：主设备发送完信息后就直接释放，从设备准备好了之后，称为主设备往原来的发送方（现在是从设备）发送准备好的数据（即把整个事务分离成两个单向过程），控制复杂，开销也挺大的

## 第七章 IO

- IO接口的功能：
    - 地址译码和设备选择
    - 实现主机与外设的通信联络控制
    - 数据缓冲
    - 信号格式的转换
    - 传送控制指令和状态信息
- IO接口当中有状态/控制寄存器，由于其传输方向相反，时间是错开的，所以合成一个寄存器
- IO **端口** 是IO接口电路中可以被CPU直接访问的寄存器
- IO端口的编址：
    - 独立编址：形成和主存地址完全独立的一套地址系统
    - 统一编址：主存地址空间划分一段出来给IO端口
- IO的方式：
    - 程序查询方式（定时轮询），下面又分两种，独占轮询，CPU就一直不停地轮询，不干别的，定时轮询，过一段时间轮询一次
    - 程序中断方式：使用中断打断CPU的任务让CPU来处理送过去的数据
        - 判断是否适合这个方式，判断处理中断的时间是否小于数据准备的时间，否则就会出现这次数据的中断还没有处理完，下一个数据又来了，就会导致数据丢失
        - 响应中断有三个条件：中断源有中断请求，CPU允许中断（异常和不可屏蔽中断除外），一条指令执行完毕（异常除外）并且没有其他更紧急的任务
        - 中断处理流程过程当中，关中断，保存断点和中断服务程序寻址都是 **硬件完成的**，之后的任务都是软件完成（中断服务程序）
        - 对于多重中断，使用屏蔽字来判断 **中断处理的优先级**，使用判优电路来判断 **中断响应的优先级。**
    - DMA方式
- DMA控制器必须要有控制总线的能力
- DMA的传送方式：
    - 停止 CPU 访存，直到DMA工作完毕，但是会导致CPU一直阻塞
    - 周期挪用：每次要传输一个字的时候都申请一次总线，传输完马上释放直到下一次
    - 交替访存：在CPU的工作周期分成两个时间片，一个时间片给CPU访存用，一个时间片给DMA用
- DMA搞完了数据传输，用中断提示CPU完成
- DMA请求优先于中断请求
- 中断有响应突发异常事件的能力，而DMA擅长连续大量数据的传输
- DMA是由硬件完成的
- 对DMA请求的响应可以在任意机器周期（取指，间址……）结束后进行
- DMA对主存的访问的优先级比CPU更高，不然就会丢数据
