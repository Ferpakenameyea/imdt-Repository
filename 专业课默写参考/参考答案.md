# 计算机硬件

## 第一章

硬件：计算机系统中实际物理装置的统称

软件：在硬件上运行的程序和相关的数据和文档

固件：将程序固化在ROM中组成的部件

一个完整的计算机系统包括：硬件系统和软件系统

冯诺依曼机的特点：

- 存储程序
- 同等地位
- 五大设备
- 二进制

存储程序：将指令以代码的形式事先输入主存，然后从首地址开始执行程序，以后就按该程序的规定顺序执行其他指令，直至程序执行结束

冯诺依曼机的基本工作方式：控制流驱动方式

现代计算机的中心：存储器

CPU区分指令和数据的依据：指令周期的不同阶段

主机：计算机硬件的主体，包括CPU和主存

计算机性能指标：

- 字长：计算机进行一次整数运算所能处理的二进制的位数
- 数据通路带宽：数据总线一次并行传送信息的位数
- 主存容量
- 运算速度：
  - 吞吐量
  - 响应时间
  - 主频
  - CPU时钟周期
  - CPI
  - MIPS
  - MFLOPS

兼容：运行在某个型号的计算机系统中的硬件/软件也能用于另一个型号的系统

软件可移植性：软件直接或进行很少的修改就能运行在另一个系列计算机上

字：用来表述被处理信息的单位

指令字长：一条指令的二进制长度

存储字长：一个存储单元的二进制长度

机器字长：计算机进行一次整数运算所能处理的二进制数据的位数

计算机体系结构：指能被程序员所见到的计算机系统的属性，包括概念性的结构与功能特性

计算机组成：实现计算机体系结构所体现的属性，包括硬件实现的具体方式

## 第二章

机器数：把符号位数字化的数

真值：机器数代表的实际值

BCD码默认为：8421码

大端：从最高有效字节到最低有效字节的顺序存储数据

## 第三章

存储系统按层次（作用）分类：

- 主存
- 辅存
- Cache

存储系统按存取方式分类：

- RAM
- ROM
- 串行访问存储器

存储器按照信息的可保存性分类：

- 易失性存储器
- 非易失性存储器

存储器的性能指标：

- 存储容量
- 存储速度
- 单位成本

Cache主存层解决什么问题：解决CPU和主存速度不匹配的问题

主存-辅存层解决什么问题：解决存储系统的容量问题

存储元：存放一个二进制位的物理器件

磁盘读写的最小单位：扇区

文件系统读写磁盘的最小单位：簇

磁盘性能指标：记录密度、容量、平均存取时间、数据传输率

RAID：Redundant Array of Independent Disks 独立冗余磁盘阵列，将多个独立的物理磁盘组成一个逻辑盘

Cache的作用：位于CPU和主存之间，存放当前CPU经常访问的指令和数据，加快CPU的访存速度

Cache集成到CPU中的好处：

- 减少对系统总线的占用，提高总线利用率
- Cache与CPU的数据通路长度缩短，访存速度加快

指令Cache和数据Cache分开的好处：

- 使取指和访存能同时执行，让流水线的实现成为可能
- 指令Cache可以用ROM实现，提高可靠性
- 数据Cache对不同类型数据的支持更加灵活

Cache写命中的策略：

- 全写法：同时修改Cache和主存中的数据
- 回写法：只在Cache块被换出时才写回主存

Cache写不命中的策略：

- 写分配法：把对应的块调入到Cache中来后进行写入
- 非写分配法：不调入Cache，只写入主存

## 第四章

ISA：Instruction Set Architecture，指令集体系结构，是计算机软硬件进行交互的核心界面，规定了处理器可以执行的指令集合，寄存器组织结构，内存访问方式以及中断处理机制等重要内容

指令：指示计算机执行某种操作的命令

指令集：一台计算机所有指令的集合

RISC的优点：

- 充分利用芯片面积
- 提高运算速度
- 便于设计，降低成本，提高可靠性
- 有利于编译程序进行代码优化
- 更好地支持流水线

## 第五章

影响CPU性能的主要技术指标：

- 频率：
  - 主频
  - 外频/前端总线频率（FSB Front Side Bus Frequency）
  - 倍频系数
- 字长
- 缓存容量
- 制程

CPU组成：运算器和控制器

指令周期：CPU从主存取出并执行一条指令的时间

机器周期：执行指令周期中一步相对完整的操作所需的时间

时钟周期：计算机主时钟的周期时间

单指令周期：所有指令都以相同时间执行

多指令周期：不同指令使用不同的执行步骤

流水线方案：指令之间并行执行

数据通路：数据在功能部件之间传送的路径

异常：由CPU内部产生的意外事件，分为故障，自陷，终止

中断：由CPU外部发送中断请求产生中断

流水线冒险：

- 结构冒险
- 数据冒险
- 控制冒险

## 第六章

总线：多个部件分时共享的公共信息传送线路

分时：同一时刻只能有一个部件向总线发信息

共享：总线上可以挂载多个部件，这些部件接收相同的信息

总线的特点：分时和共享

总线的分类：

- 片内总线
- 系统总线
  - 数据总线
  - 地址总线
  - 控制总线
- 通信总线
- IO总线

总线的性能指标：

- 传输周期
- 时钟周期
- 工作频率
- 时钟频率
- 宽度
- 带宽
- 复用

总线的优点：

- 简化系统结构，便于设计制造，提高可靠性
- 便于接口设计
- 便于系统扩充
- 便于软件设计
- 便于故障诊断

总线的问题和解决方法：

- 多个设备同时要求使用总线时，会导致总线冲突
- 总线仲裁部件

总线主设备：一次总线传输期间，拥有总线控制权的设备

总线从设备：一次总线传输期间，配合主设备完成传输的设备

数据传输进行同步的两种方式：

- 同步传输
- 异步传输

总线定时方式：

- 同步定时方式
- 异步定时方式
  - 不互锁：请求方过一段时间后自动撤销
  - 半互锁：请求方收到从设备回答信号后才撤销，从设备过一段时间后自动撤销
  - 全互锁：请求方收到从设备回答信号后才撤销，从设备必须知道请求方撤销请求后才撤销回复信号。
- 半同步定时方式：使用 Wait 信号线轮询
- 分离式定时方式

## 第七章

IO编址方式：

- 统一编址
- 独立编址

IO控制方式：

- 程序查询方式
- 程序中断方式
- DMA方式
- 通道方式

IO接口的功能：

- 地址译码和设备选择
- 数据缓冲
- 信号格式转换
- 主机和外设的通信联络控制
- 传送控制命令和状态信息

接口：在主机与外部设备间传输数据时进行协调工作的逻辑部件

接口电路：实现接口的物理电路

端口：接口电路中可以被CPU直接访问的寄存器

中断隐指令：CPU响应中断后转去执行中断服务程序前的一系列操作，包括关中断，保存断点，中断服务程序寻址

中断隐指令实现职责由谁承担（硬件/软件）：硬件

中断向量：中断服务程序的入口地址

中断向量表：存放中断向量的存储区

中断向量地址：中断向量所在的地址

向量中断：一种识别中断源的方式

INTR：Interruption Request 中断请求触发器

EINT：Enable Interruption 中断允许触发器

中断处理优先级：当一个中断正在处理时另一个中断到来时要不要转去处理它，由中断屏蔽字寄存器决定

中断响应优先级：当多个中断同时到来时先处理谁，由中断判优电路决定

DMA的传输方式：

- 停止CPU访存
- 周期挪用
- 交替访存
